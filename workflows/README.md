# DraftTool Workflows

Use this file for common maintenance tasks instead of reconstructing commands
from chat history.

## Project Shape

- DraftTool is a standalone Flutter app in this repository.
- The app shares the Pokoin Firebase project for Auth, usernames, balances, and
  tournament ticket escrow.
- Do not merge DraftTool screens into the CardVault/Pokoin Flutter app. The
  intended web entry point is a standalone deployment such as
  `makepair.pokoin.com`.
- The existing Pokoin site should only redirect `pokoin.com/makepair` to the
  standalone deployment when the web app is ready.

## Local Development

Create local env values:

```bash
cp .env.example .env.local
```

Run web locally:

```bash
flutter run -d chrome --web-port 5175 $(./tool/flutter_env.sh)
```

Run checks:

```bash
flutter analyze
flutter test
npm run check
```

## Assets

- Pokoin brand art is local at `assets/brand/pokoin.svg`.
- Keep the app self-contained; do not depend on remote brand images for core UI.
- If the upstream Pokoin logo changes, copy the updated asset from the
  CardVault/Pokoin project and rerun `flutter analyze` and `flutter test`.

## Firebase And Secrets

- `.env.local` is intentionally ignored. Never commit real env files.
- `lib/firebase_options.dart` reads client config from Dart defines generated by
  `tool/flutter_env.sh`.
- Firebase web client config is public configuration, not a Firebase Admin
  secret, but keep it out of source to reduce accidental exposure.
- Server-only Firebase Admin env vars belong in Vercel project settings or an
  untracked local env file.

## Web Deployment

Build the standalone web output:

```bash
./deploy-drafttool-web.sh
```

This creates a Flutter web release, copies `api/*.js` handlers and Vercel
metadata into `build/web`, and leaves the output ready for the standalone
DraftTool Vercel project.

## Tournament Flow

- New events default to `BO1 + Top Cut` for faster Swiss rounds and a playoff
  finish. `BO3` and `BO3 + Top Cut` remain available for longer events.
- Top-cut events do not end when Swiss rounds are complete. Use `Start top cut`
  to seed the elimination bracket from final Swiss standings, then report top
  cut matches until the winner is finalized.
- Top-cut size is proportional to starting attendance: Top 2 for very small
  events, Top 4 for events under 17 players, and Top 8 for larger events.
- During Swiss result entry, use the `x` button next to a player to drop them.
  Dropped players remain in historical standings but are excluded from future
  Swiss pairings, effectively giving them losses by absence from later rounds.
